Better Toast

Types

You can customize the type of toast you want to render, and pass an options object as the second argument.

Headless

Go fully custom by passing a standalone component to the headless() method. Your component owns the visuals: no default styles, but stacking, positioning, and enter/leave animations still come from the toaster.

Position

Enter and exit animations change depending on the position.

@for (pos of positions; track pos) { }
<app-toaster position="{{ toasterPosition() }}" />

Offset

Viewport inset in pixels for the toast stack. Default desktop offset is 24px and default mobile offset is 16px. You can customize offsets through offset and mobileOffset inputs.

Desktop offset

@for (side of offsetSides; track side) {
@switch (side) { @case ('top') { {{ offsetTopPx() }} px } @case ('right') { {{ offsetRightPx() }} px } @case ('bottom') { {{ offsetBottomPx() }} px } @case ('left') { {{ offsetLeftPx() }} px } }
}

Mobile offset

@for (side of offsetSides; track side) {
@switch (side) { @case ('top') { {{ mobileOffsetTopPx() }} px } @case ('right') { {{ mobileOffsetRightPx() }} px } @case ('bottom') { {{ mobileOffsetBottomPx() }} px } @case ('left') { {{ mobileOffsetLeftPx() }} px } }
}
<app-toaster [offset]="{ top: '{{ offsetTopPx() }}px', right: '{{ offsetRightPx() }}px', bottom: '{{ offsetBottomPx() }}px', left: '{{ offsetLeftPx() }}px' }" [mobileOffset]="{ top: '{{ mobileOffsetTopPx() }}px', right: '{{ mobileOffsetRightPx() }}px', bottom: '{{ mobileOffsetBottomPx() }}px', left: '{{ mobileOffsetLeftPx() }}px' }" />

Duration

Sets the auto-dismiss duration for toasts. The default is 4000ms. Use Infinity to keep toasts until manually dismissed.

@if (!isManualDismissDuration()) { {{ (toasterDurationMs() / 1000).toFixed(1) }} s }
<app-toaster @if (isManualDismissDuration()) { duration="Infinity" } @else { [duration]="{{ toasterDurationMs() }}" } />

Rich Colors

When true, success/error/info/warning use semantic background and border colors.

<app-toaster @if (richColors()) { [richColors]="true" } />

Close button

When off, the per-toast dismiss control is hidden. Toasts still auto-dismiss or can be cleared with the service.

<app-toaster @if (closeButton()) { [closeButton]="true" } />

Programmatic dismiss

show(), success(), error(), info(), warning(), custom(), loading() return a toast id. Pass it to dismiss(id) to remove one toast, or call clear() to remove all.

const id = this.toaster.show('Hello'); this.toaster.dismiss(id); this.toaster.clear();
